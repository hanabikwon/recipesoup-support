  ---
  🎯 기획 의도

  왜 이 앱을 만드는가?

  개인적 동기

  "혼자 사는 30대로서, 매일 저녁 '뭐 먹지?' 고민하며 대충 때우는 나를 보며 시작한 프로젝트"

  - 문제 인식: 혼자 먹는 밥은 대충, SNS용 요리는 정성껏
  - 해결 방향: 나를 위한 요리도 의미 있게 기록할 수 있는 공간
  - 차별화: 레시피가 아닌 '감정'을 기록하는 유일한 앱

  시장 기회

  타겟 시장:
    - 1인 가구 700만 (2024년 기준)
    - 3040 여성 1인 가구 150만
    - 요리 앱 사용자 중 40%가 기록 니즈

  경쟁 공백:
    - 모든 레시피 앱: 공유/소셜 중심
    - 일기 앱: 요리 특화 기능 없음
    - SNS: 진솔한 기록 불가능
    → "감정 기반 프라이빗 요리 일기" 포지션 비어있음

  개발자로서의 강점

  - Flutter 크로스플랫폼으로 iOS/Android 동시 개발
  - OpenAI API 활용한 차별화 기능
  - 1인 개발의 빠른 의사결정과 실행

  ---
  👤 타겟 사용자

  Primary Target: "나 같은 사람"

  3040 혼자 사는 직장인 여성

  핵심 특징

  라이프스타일:
    - 평일: 회사-집 반복
    - 주말: 나를 위한 시간
    - 요리: 스트레스 해소 수단

  디지털 행동:
    - SNS 피로감 느낌
    - 프라이버시 중시
    - 간편한 기록 선호

  핵심 니즈:
    "나를 위한 요리를 의미 있게 기록하고 싶다"
    "진짜 내 감정을 숨김없이 쓰고 싶다"
    "혼밥도 특별하게 만들고 싶다"

  사용 시나리오

  메인 시나리오: 퇴근 후 저녁 (70%)

  19:30 퇴근 → 마트 → 집
  20:00 요리 시작
  20:30 완성
  20:35 Recipesoup 오픈
    → 사진 찍기
    → "오늘도 수고한 나를 위한 저녁"
    → 😌 평온 선택
    → #혼밥 #나를위한한끼
  20:38 저장 완료 (3분 소요)

  서브 시나리오: 주말 브런치 (30%)

  AI 사진 분석 활용
    → 사진만 찍고
    → AI가 레시피 자동 생성
    → 감정 메모만 추가
    → 1분 내 완료

  💡 핵심 컨셉

  서비스 정의

  "요리에 담긴 감정을 기록하는 나만의 비밀 일기"

  Core Value

  1. Privacy First: 로그인 없음, 100% 로컬 저장
  2. Emotion Focused: 레시피보다 감정이 중요
  3. AI Powered: 10초 만에 레시피 생성
  4. For Singles: 1인분 기본, 혼밥 긍정

  슬로건

  "나를 위한 한 끼도 특별하게"

  🚀 MVP 범위 (1인 개발 8주)

  핵심 기능만 (Less is More)

  ✅ 포함 (MUST)

  1. 감정 레시피 작성 (핵심)
     - 사진 + 감정 메모 + Mood
     - 수동 입력 폼

  2. AI 사진 분석 (차별화)
     - OpenAI Vision API
     - 자동 레시피 생성

  3. 목록/상세 보기
     - 시간순 정렬
     - 카드형 리스트

  4. 로컬 저장
     - Hive DB
     - 로그인 없음

  ❌ 제외 (추후)

  - 검색/필터 (v2)
  - 통계 분석 (v2)
  - URL 스크래핑 (v2)
  - 키워드 생성 (v2)
  - 폴더 정리 (v2)
  - 설정 화면 (v2)

  화면 구성 (6개만)

  1. 스플래시 (자동)
  2. 홈 (리스트)
  3. 작성 (수동)
  4. AI 분석 (사진)
  5. 상세보기
  6. 빈 상태
